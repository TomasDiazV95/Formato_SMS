Objetivo general:
Migrar la interfaz actual en Flask a un front React (Vite + Tailwind) para SMS, IVR y Mail, manteniendo los endpoints existentes y preparando una futura administración de CAMPO1 en MySQL.

Trabajo realizado hasta ahora:
1. Backend: se agregó un mapa de alias de columnas en services/constants.py y mejoras en los constructores de plantillas (services/mail_templates.py) para detectar columnas con flexibilidad. Se mantiene el uso de dtype=str al leer Excel para conservar los valores completos de las OP.
2. React: se creó la app react-frontend (Vite + React Router + Tailwind 3.x). Configuración base en src/App.jsx, src/main.jsx, Tailwind, router y utilidades.
3. Portal React: páginas Portal, Procesos, Cargas y Reportes con navegación básica.
4. Módulo SMS React: página completa (src/pages/sms/SmsPage.jsx) con formularios Masivo y CRM, manejo de ejemplos, alertas y descargas desde /sms/athenas y /sms/crm.
5. Módulo IVR React: estructura creada (src/pages/ivr/IvrPage.jsx). Se corrigieron etiquetas JSX abiertas y se completó el formulario CRM. Ambos formularios (Athenas y CRM) permiten subir archivos, seleccionar mandante/CAMPO1, definir fecha/horario/intervalo y descargar resultados.
6. Build: npm run build dentro de react-frontend compila sin errores.

Pendiente / próximos pasos:
1. Confirmar lógica específica de IVR (mapeos de mandantes, reenviadores) que ya existe en helpers del backend.
2. Implementar módulo Mail React reutilizando helpers de services/mail_templates.py.
3. Planificar panel “Gestionar CAMPO1” con persistencia en MySQL (reemplazar localStorage).
